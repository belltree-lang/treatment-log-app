# Issue #1160 対応サマリー（共有用）

## 変更点
- 銀行引落シートのフラグ列（未回収/合算/オンライン）の順序を常に **AE=未回収 → AF=合算 → AG=オンライン** に揃える処理を追加。
- テンプレートから銀行引落シートを生成・更新する際に、フラグ列のヘッダとチェックボックス列を再整備するように調整。
- 列解決のヘッダ判定で共有定数（合算ヘッダ）を使用し、文字列の食い違いを防止。

## 変更理由
- AE/AF/AG の列意味と順序がずれると、既存のチェック状態の意味が逆転し、再集計やテンプレート更新時にフラグが誤解釈されるため。
- 既存チェック状態を維持しつつ、列の意味と順序が崩れないようにする必要があるため。

## 影響範囲
- 対象は **銀行引落シートの列ヘッダ・チェックボックス列・列整列ロジック** のみ。
- 金額算出・請求/領収ロジック・PDF テンプレートには変更なし。
