# 共有用サマリー（Issue #1262）

## 変更点
- 自費請求の表示条件を「施術録の施術時間区分（H列）」ベースの回数集計に変更し、金額・entries・manualSelfPayAmount・オンライン同意フラグ等の金額系ロジックに依存しない判定へ統一。  
- 請求生成時に `selfPayVisitCount` を各行へ付与し、既存互換のため `selfPayCount` にも同値を格納。  
- UIでは `selfPayVisitCount >= 1` を満たす患者のみ【自費請求】へ表示。  

## 理由
- 施術録の「施術時間区分」を唯一の自費回数判定ソースとする仕様変更に対応し、金額やentriesにより表示が揺れる問題を防止するため。  

## 影響範囲
- **請求生成ロジック**: 施術録のカテゴリ集計から自費回数を算出し、請求JSONへ追加。  
- **表示ロジック**: 【自費請求】セクションの表示判定が回数ベースへ変更。  
- **既存の保険回数・金額ロジック**: 影響なし（従来通り維持）。
