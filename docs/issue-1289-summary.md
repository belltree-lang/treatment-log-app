# Issue 1289 対応サマリー（共有用）

## 変更点
- 保険セクションの回数表示・表示判定・施術料表示が同じ値を参照するよう、保険回数の統一算出ロジックを追加。
- `insuranceCount` と `mixedCount` を合算して保険回数を求め、値が無い場合のみ `visitCount` にフォールバック。
- 施術料表示も同じ保険回数を用いて再計算するように調整。

## 理由
- `visitCount` / `insuranceCount` / `mixedCount` を個別に参照していたため、表示判定と施術料が食い違い「交通費のみ表示される」ケースが発生していた。
- UI 内の参照値を統一することで、表示の一貫性と誤表示防止を図る。

## 影響範囲
- 影響は **renderBillingResult 内の保険 UI 表示のみ**。
- 集計ロジック、PDF 生成、銀行連携ロジックには変更なし。
