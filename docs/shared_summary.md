# 共有用サマリー

## 変更点
- 請求一覧の「合計（grandTotal）」を手入力で編集できるようにし、編集値を BillingOverrides に保存できるようにしました。
- BillingOverrides の読み込み・保存に `manualBillingAmount` を追加し、上書きがある場合は最終請求額として `grandTotal` に反映するようにしました。
- 銀行引落シート作成や PDF 生成で参照される請求額が、上書き済みの最終請求額になるように揃えました。

## 理由
- 集計ロジックは変更せず、集計値を初期値として扱い、手入力の請求額を最終値として扱う要件に対応するためです。
- 既存の BillingOverrides 仕組みを活用し、月次・患者単位の上書きを保存できるようにするためです。

## 影響範囲
- 請求一覧 UI（合計の編集と上書き表示）
- BillingOverrides の保存・読み込み
- 銀行引落シート作成、請求 PDF 生成で使用される最終請求額
